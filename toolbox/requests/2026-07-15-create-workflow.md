# 크리에이트 자동 워크플로 작업 기록

## 확정 입력

- 작업 종류: 서로 중복되지 않는 신규 공개 도구 5개 제작
- 대상: 안전 비밀번호 생성기, 이미지 크기·용량 변환기, 색상 변환·대비 검사기, Unix 시간 변환기, 글자수·텍스트 정리기
- 버전 단계: 각 도구 최초 `0.0.1v`
- Blogger 게시: 완료된 5개 모두 공개
- Git·배포: `main` 커밋·푸시와 GitHub Pages 공개 확인
- 종료 예약: 사용자의 `아니` 응답에 따라 사용하지 않음

## 공통 구현 원칙

- 기능별 고정 iframe 진입점과 독립 `tool.js` 모듈을 생성 시점부터 사용한다.
- 공통 스타일, KOR / ENG, 클립보드 차단 대체 UI, 모바일 자동 포커스 방지와 높이 동기화를 재사용한다.
- 도구 정의, 상세 설명, 사용자 FAQ, 해시태그, 20~30개 검색 키워드, 누적 패치노트, 카탈로그와 AI 색인을 함께 생성한다.
- 기능을 브라우저 안에서 처리하고 입력이나 파일을 서버에 저장하지 않는다.

## 워크플로 토큰 최적화

- 변경 영향도 규칙으로 필요한 검사만 선택한다.
- `select → implement → validate → publish → commit → deploy → confirm` 단계 상태를 기록해 최초 미완료 단계부터 재개한다.
- 데일리 라이트·하드·올인·크리에이트 실행 수량과 완료 조건을 JSON 실행표로 분리한다.
- 반복 차단 사유는 코드와 다음 행동으로 표준화한다.
- 스킬에는 실행 순서만 두고 세부 정책은 저장소 문서와 기계 판독 JSON을 단일 원본으로 사용한다.

## 공통 회귀 방지

- `toolbox/quality/known-issues.json`에 실제로 재현된 결함의 증상, 원인, 수정 계약과 반복 검사를 코드로 기록한다.
- 변경 파일과 일치하는 이슈만 읽는 `toolbox-regression-guard` 스킬을 사용해 동일 문제의 재발을 확인한다.
- 이슈마다 새 Markdown이나 스킬을 만들지 않고 단일 JSON 레지스트리와 단일 얇은 스킬을 유지한다.
