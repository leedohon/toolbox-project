# 놀이 도구 다중 목록 입력 UX 개선

## 사용자 지시

- 줄바꿈 텍스트로 참가자나 항목을 한꺼번에 입력하는 방식을 개별 입력 행 구조로 바꾼다.
- `+` 버튼으로 항목 또는 사람을 추가할 수 있게 한다.
- 비어 있는 칸은 사용자가 바로 이해할 수 있는 세련된 힌트를 제공한다.
- 이후 가변 길이 목록을 입력하는 모든 도구에 유사한 UX를 적용한다.
- UX 기반 디자인 준수를 전역 Markdown 지침에 기록한다.

## 적용 대상

1. 사다리게임 `0.1.0v`
   - 참가자와 결과를 같은 반복 행에 배치한다.
   - `+ 참가자와 결과 추가`, 행별 삭제, 번호를 제공한다.
2. 돌림판 `0.1.0v`
   - 후보를 개별 입력 행으로 제공한다.
   - `+ 항목 추가`, 행별 삭제, 번호를 제공한다.
3. 다중팀 편가르기 `0.1.0v`
   - 참가자를 개별 입력 행으로 제공한다.
   - `+ 참가자 추가`, 행별 삭제, 번호를 제공한다.
4. 숫자뽑기
   - 가변 목록 입력이 아니므로 현재 숫자 필드 구조를 유지한다.

## 공통 상호작용

- 최소 두 행을 유지하고 최대 개수에 도달하면 추가 버튼을 비활성화한다.
- 마지막 입력에서 Enter를 누르면 새 행을 만들고 바로 포커스를 이동한다.
- 빈 행이 있으면 입력을 실행하지 않고 해당 행에 `이 칸을 입력해 주세요.` 힌트와 오류 상태를 표시한다.
- 입력을 시작하면 해당 행의 오류 상태를 즉시 해제한다.
- 삭제 후 번호와 접근성 라벨을 다시 계산한다.
- 공통 목록 편집 동작은 `assets/play-tools.js`, 공통 외형은 `assets/play-tools.css`에서 관리한다.

## 검증

- 모바일과 데스크톱에서 추가, 삭제, Enter 추가, 빈칸 오류, 최대 개수, 긴 입력과 버튼 줄바꿈을 확인한다.
- 사다리 참가자·결과 쌍이 항상 같은 수로 유지되는지 확인한다.
- 돌림판 회전과 편가르기 균등 배분의 기존 핵심 기능이 유지되는지 확인한다.
- 공개 게시글의 iframe 높이, 가로 넘침과 내부 스크롤을 다시 확인한다.
