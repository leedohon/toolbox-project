URL 변환 기능이 통합되었습니다

URL 인코딩·디코딩은 이제 텍스트 인코더·디코더에서 Base64와 함께 사용할 수 있습니다.

  • Base64 · URL 형식 선택
  • URL 구성요소 · 전체 주소 처리
  • 입력·결과 바꾸기와 결과 복사
  • 모바일 반응형 화면

통합 도구 사용하기

도구가 보이지 않으면 통합 텍스트 인코더·디코더 게시글 열기를 이용하세요.

사용 방법

  1. 변환 형식에서 URL을 선택합니다.
  2. 인코딩·디코딩과 쿼리 값 또는 전체 주소 범위를 고릅니다.
  3. 내용을 입력해 변환한 뒤 결과를 복사합니다.

URL 인코더·디코더 상세 설명

URL 인코딩은 공백, 한글, 특수문자를 주소에서 안전하게 전달할 수 있는 퍼센트 문자열로 바꾸는 과정입니다. 쿼리 값만 변환할 때와 주소 전체를 변환할 때 유지해야 하는 예약문자가 다릅니다.

이 도구는 쿼리 값·텍스트와 전체 URL 모드를 나누어 제공하며 인코딩 결과를 다시 원문으로 디코딩할 수 있습니다. 모든 변환은 현재 브라우저에서 처리됩니다.

자주 묻는 질문

쿼리 값과 전체 URL 모드는 어떻게 다른가요?

쿼리 값 모드는 ?, &, = 같은 예약문자도 변환하고, 전체 URL 모드는 : / ? & =처럼 주소 구조에 필요한 문자를 유지합니다.

한글 주소도 인코딩할 수 있나요?

네. 한글과 공백을 UTF-8 기준 퍼센트 문자열로 변환할 수 있습니다.

디코딩 오류가 나면 무엇을 확인해야 하나요?

퍼센트 기호 뒤에 두 자리 16진수가 있는지, 문자열이 중간에서 잘리지 않았는지 확인해 주세요.

입력 내용이 서버로 전송되나요?

아니요. 인코딩과 디코딩은 현재 브라우저 안에서만 실행됩니다.
