# 다용도 뽑기게임 통폐합

## 요청

- 사다리게임, 돌림판, 숫자뽑기, 편가르기를 하나의 기능과 하나의 게시글로 통합한다.
- 새 카테고리는 `다용도 펀박스`, 공개 도구명은 `다용도 뽑기게임`으로 사용한다.
- 공개 명칭에서 `다중팀`을 제거하고 `편가르기`로 표시한다.
- 폴더와 모듈 구조는 새 기능을 만들 때부터 다른 통합 화면에 재사용하기 쉽게 설계한다.
- 기존 프로젝트 전체를 소급 변경하지 않고 이번 통합과 이후 신규·통합 기능부터 순차 적용한다.

## 구조 결정

- 공개 진입점은 `embed/multipurpose-draw-game/index.html` 하나로 고정한다.
- `embed/multipurpose-draw-game/modules.json`을 모듈 등록 단일 원본으로 사용한다.
- 등록 파일은 기능 식별자, 한·영 표시명, 실행 경로, 높이 메시지 식별자와 기능 지원 범위를 분리한다.
- 통합 화면은 등록 파일을 읽어 모드 선택 UI를 만들고, 개별 기능은 독립 iframe 모듈로 실행한다.
- 이번에는 검증된 기존 네 엔진을 호환 모듈로 등록해 기존 결과 코드가 계속 열린다. 이후 신규 모듈은 처음부터 같은 계약을 구현한다.
- 기존 네 도구는 삭제하지 않고 `status: retired`, `replacementTool: multipurpose-draw-game`으로 카탈로그에서 제외한다.
- 기존 index, 버전 폴더, 게시글 URL과 공유 결과 코드는 보존하며 기존 게시글은 새 통합 게시글 안내로 전환한다.
- 새 도구에는 사용하지 않은 새 index `11`을 배정하며 은퇴 index 4~7은 재사용하지 않는다.

## 검증

- 네 모드가 한 화면에서 전환되고 전환 후 입력 상태가 유지되는지 확인한다.
- 모듈 등록 JSON과 메시지 식별자, 실행 경로를 자동 검증한다.
- KOR / ENG, 반복 입력, 오류 안내, 결과 코드, PNG 저장과 클립보드 대체 복사가 각 모드에서 유지되는지 확인한다.
- PC와 모바일에서 모드 선택, iframe 높이, 가로 넘침과 버튼 줄바꿈을 확인한다.
- 카탈로그에는 다용도 뽑기게임 하나만 표시되고 기존 네 주소는 새 게시글을 안내하는지 확인한다.

## AI 빠른 색인 추가 요청

- 루트, toolbox 작업 폴더와 신규·수정 도구 폴더에 AI가 기능 위치와 관계를 빠르게 찾는 `ai-index.json`을 둔다.
- 중앙 색인은 도구 정의, 게시글 원본, 버전 manifest, 실행 진입점, 모듈 등록, 공개 상태와 대체 도구 관계를 제공한다.
- 모든 기존 도구에 수동 소급하지 않고 신규·수정 도구의 색인부터 순차 생성한다.
- 색인은 손으로 중복 관리하지 않고 생성 스크립트로 갱신한다.
