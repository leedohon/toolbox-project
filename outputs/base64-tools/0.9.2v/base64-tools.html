텍스트 인코더·디코더

Base64, URL, ASCII, Unicode와 UTF-8 바이트 코드를 한 화면에서 변환하세요.

  • 6가지 텍스트 코드 형식
  • ASCII · Unicode 코드 변환
  • UTF-8 16진수 · 2진수 변환
  • 입력·결과 바꾸기와 KOR / ENG

텍스트 변환하기

도구가 보이지 않으면 새 창에서 열기를 이용하세요.

사용 방법

  1. Base64, URL, ASCII, Unicode, 16진수, 2진수 중 형식을 선택합니다.
  2. 인코딩 또는 디코딩을 고르고 원문이나 코드를 입력합니다.
  3. 변환 결과를 복사하거나 반대 변환으로 바로 확인합니다.

텍스트 인코더·디코더 상세 설명

Base64와 URL뿐 아니라 ASCII 10진 코드, Unicode 코드 포인트, UTF-8 16진수와 2진수 변환을 한 화면에 통합했습니다. 같은 입력·결과 흐름에서 형식만 선택하므로 개발 문서, 문자 코드, 데이터 확인 작업을 빠르게 이어갈 수 있습니다.

Base64는 텍스트나 파일의 바이트를 영문자와 숫자로 이루어진 문자열로 표현하는 방식입니다. 이메일 본문, 데이터 URL, API 요청처럼 바이너리 데이터를 텍스트로 전달해야 할 때 주로 사용합니다.

ASCII는 영문과 숫자 같은 기본 문자에 적합하고 Unicode 코드 포인트는 한글과 이모지를 포함한 다양한 문자를 정확하게 확인할 때 사용합니다. 16진수와 2진수는 UTF-8 바이트 기준으로 표시합니다.

이 도구는 일반·URL-safe Base64와 파일의 data URL 변환도 지원합니다. 모든 텍스트와 파일 변환은 현재 브라우저 안에서 진행되므로 입력한 내용이 별도 서버로 전송되지 않습니다.

자주 묻는 질문

어떤 텍스트 변환 형식을 지원하나요?

Base64, URL, ASCII 10진 코드, Unicode 코드 포인트, UTF-8 16진수와 UTF-8 2진수를 지원합니다.

ASCII에서 한글이나 이모지를 변환할 수 없다고 나옵니다.

ASCII는 0부터 127까지의 기본 영문, 숫자, 기호만 표현합니다. 한글이나 이모지는 Unicode 코드 포인트 또는 UTF-8 16진수·2진수를 선택하세요.

Unicode 코드는 어떤 형식으로 입력하나요?

U+AC00 U+B098처럼 U+와 16진수 코드 포인트를 공백으로 구분해 입력합니다. 이모지처럼 큰 코드 포인트도 지원합니다.

16진수와 2진수는 어떤 기준인가요?

문자열을 UTF-8 바이트로 바꾼 값을 표시합니다. 16진수는 두 자리, 2진수는 여덟 자리 단위로 구분합니다.

Base64와 URL 변환은 어떻게 바꾸나요?

화면 위쪽의 변환 형식에서 Base64 또는 URL을 선택하세요. URL을 선택하면 쿼리 값·문장과 주소 전체 중 처리 범위도 고를 수 있습니다.

URL-safe Base64는 언제 사용하나요?

주소, 쿼리 값, 파일명처럼 +와 / 문자가 특별한 의미를 가질 수 있는 곳에서 사용합니다. URL-safe 형식은 +를 -, /를 _로 바꾸고 끝의 = 패딩을 생략합니다.

일반 Base64와 URL-safe Base64를 모두 디코딩할 수 있나요?

네. 디코딩할 때 두 형식을 자동으로 인식하므로 별도 옵션을 바꾸지 않아도 됩니다.

파일이 서버로 전송되나요?

아니요. 텍스트와 파일은 현재 브라우저 안에서만 읽고 변환합니다.

Base64가 암호화 방식인가요?

아닙니다. 누구나 원문으로 되돌릴 수 있는 표현 방식이므로 비밀번호나 개인정보를 보호하는 용도로 사용하면 안 됩니다.
