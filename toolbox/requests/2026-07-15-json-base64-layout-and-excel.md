# JSON · Base64 레이아웃 및 Excel 기능 작업 지시

## 확정 요구사항

1. Base64 도구의 원본 텍스트와 변환 결과를 좌우가 아닌 위아래 한 열로 배치한다.
2. JSON 도구의 입력과 결과·오류 위치를 좌우가 아닌 위아래 한 열로 배치한다.
3. 두 도구는 모바일과 PC에서 iframe의 사용 가능한 가로 폭을 자연스럽게 채우고, 비정상적으로 좁아지는 고정 폭·중복 최대 폭을 제거한다.
4. JSON 도구에 Excel 표 미리보기와 실제 `.xlsx` 다운로드를 추가한다.
5. 변환, 표 생성, 파일 생성은 브라우저 안에서만 처리한다.

## JSON → Excel 변환 규칙

- 객체 배열은 배열 항목 하나를 한 행으로 변환한다.
- 객체 배열 안의 중첩 객체는 `user.name` 같은 점 표기 열 이름으로 펼친다.
- 일반 객체는 `키`, `값` 두 열로 표시하고 중첩 객체 키는 점 표기로 펼친다.
- 중첩 배열과 복합 배열 값은 JSON 문자열로 한 셀에 저장한다.
- 원시값 배열은 `값` 한 열로 표시한다.
- 화면 미리보기는 최대 100행을 표시하되 Excel 파일에는 전체 데이터를 기록한다.

## 구현 및 검증

- JSON Excel 파일은 SheetJS Community Edition 공식 브라우저 배포본을 사용해 실제 `.xlsx`로 생성한다.
- 입력·결과·표 영역에는 `min-width: 0`, `width: 100%`와 필요한 가로 스크롤을 적용해 부모 폭을 넘기거나 강제로 축소되지 않게 한다.
- 좁은 모바일 화면에서는 버튼과 입력 영역이 한 열 또는 자연스러운 줄바꿈으로 표시되어야 한다.
- 데스크톱과 모바일 폭에서 입력, 결과, Excel 표, 버튼, iframe 높이를 확인한다.
- 변경 후 도구 정의, 버전 기록, 카탈로그, Blogger 게시글, Git 커밋·푸시, GitHub Pages 배포를 함께 갱신한다.
