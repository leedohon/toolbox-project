# 이미지 모자이크 도구 신규 게시글

## 사용자 지시

- 기존 게시글의 부가 기능이 아니라 독립된 새 도구 게시글 하나를 추가한다.
- 많은 사용자가 쉽게 활용할 수 있는 기능을 우선한다.
- 예시 중 이미지 위를 드래그해 모자이크를 넣는 도구를 구현한다.
- 인증서 암호화·복호화 도구는 안전한 목적과 인증서 형식이 확정되지 않았으므로 이번 범위에서 제외한다.

## 구현 결정

- 도구명: 이미지 모자이크 도구
- 식별자: `image-mosaic-tool`
- 지원 파일: PNG, JPEG, WebP
- 입력 방식: 파일 선택 및 드래그 앤 드롭
- 편집 방식: 이미지 위에서 마우스 또는 손가락을 드래그해 원형 브러시 영역을 모자이크 처리
- 설정: 브러시 크기와 모자이크 강도 슬라이더
- 보조 기능: 한 단계 되돌리기, 원본으로 초기화, PNG 저장
- 체험 기능: 파일 선택 전에도 편집 방식을 확인할 수 있는 샘플 이미지
- 개인정보 보호: 이미지는 서버로 전송하지 않고 현재 브라우저에서만 처리

## 입력 제한과 안전

- 파일 크기는 15MB 이하로 제한한다.
- 이미지 한 변은 5,000px 이하, 전체 픽셀은 20,000,000 이하로 제한한다.
- PNG, JPEG, WebP가 아닌 파일은 처리하지 않는다.
- 편집 이력은 이미지 사본이 아니라 브러시 좌표 목록으로 저장해 메모리 사용량을 제한한다.
- 한 번에 최대 100개 브러시 동작을 보관한다.

## UI 기준

- 기존 화이트·스카이블루 토큰과 버튼 형태를 사용한다.
- 이미지가 없을 때는 드롭 영역과 짧은 안내만 표시한다.
- 모바일에서도 파일 선택, 손가락 드래그, 되돌리기, 저장을 모두 사용할 수 있어야 한다.
- 캔버스는 원본 해상도로 처리하되 화면 폭에 맞게 축소 표시한다.
- 이미지가 화면을 가로로 넘거나 게시글 iframe에 내부 스크롤을 만들지 않게 한다.

## 릴리스 및 게시 명령

1. `node scripts/build-tool-catalog.mjs`
2. `node scripts/validate-tool-releases.mjs`
3. `node scripts/blogger-publish-tool-posts.mjs image-mosaic-tool`
4. `node scripts/blogger-sync-post-urls.mjs`
5. 관련 변경을 커밋해 `main`에 푸시하고 GitHub Pages 및 공개 Blogger 게시글을 확인한다.

## 디자인 검수

- 모바일 390px 및 데스크톱에서 파일 선택과 드래그 앤 드롭 UI를 확인한다.
- 실제 이미지를 불러와 드래그 편집, 슬라이더, 되돌리기, 초기화, PNG 저장을 확인한다.
- 잘못된 파일, 제한 초과, 이미지가 없는 상태의 오류 안내를 확인한다.
- 가로 넘침, 캔버스 잘림, 버튼 줄바꿈, 비정상적인 여백을 수정 후 재검수한다.
