# 블로그 UI 복구 및 실용 기능 3종 자율 작업

## 사용자 지시

- 공개 블로그와 게시글을 직접 확인해 깨진 UI를 수정한다.
- 작업에 필요한 테마 변경은 저장소의 전역 CSS·JavaScript에 반영하고 결과를 보고한다.
- 기존 공개 도구에 실용적인 기능 약 3개를 선정해 구현하고 게시글 설명까지 갱신해 공개한다.
- 구현 후 데스크톱·모바일 디자인 검증, Blogger 반영, 커밋·푸시, GitHub Pages 배포 확인까지 자율 처리한다.

## 공개 화면 진단

1. 모바일 375px 화면에서 Blogger 기본 열 여백과 게시글 패딩이 중복되어 실제 iframe 폭이 약 235px까지 축소된다.
2. Blogger 기본 검색 위젯이 전역 테마와 어울리지 않는 표 형태로 노출된다.
3. 홈 화면의 검색 버튼 폭이 좁아 글자가 두 줄로 갈라진다.
4. 홈의 맞춤 소개와 Blogger 기본 헤더가 중복되어 정보 위계가 흐려진다.
5. JSON과 Base64 게시글 제목의 접두어 뒤 공백이 일관되지 않는다.

## 전역 테마 변경

- 모바일에서 `.column-center-inner`와 `#main`의 중복 좌우 여백을 제거한다.
- 모바일 게시글 패딩과 공통 도구 카드 패딩을 줄여 iframe 사용 폭을 확보한다.
- 기본 Blogger 검색 탭을 숨기고 홈의 맞춤 도구 검색을 단일 검색 UI로 사용한다.
- 홈에서는 중복 Blogger 헤더를 숨기고 맞춤 소개를 대표 헤더로 사용한다.
- 검색 버튼은 한 줄 최소 폭을 유지한다.
- 게시글 제목은 `[초간단 툴박스] <도구 제목>` 형식으로 통일한다.

## 추가 기능 3종

1. JSON 도구: 현재 Excel 표 데이터를 UTF-8 CSV 파일로도 다운로드한다.
2. Base64 도구: 텍스트 인코딩 시 URL-safe Base64 형식(`-`, `_`, 패딩 제거)을 선택하고 URL-safe 문자열을 자동 디코딩한다.
3. QR · 바코드 생성기: 전경색과 배경색을 선택해 QR 코드와 Code 128 바코드에 적용한다.

## 검증 상태

- 홈, JSON, Base64, QR 게시글을 데스크톱·모바일에서 확인한다.
- 빈 상태, 정상 입력, 오류 입력, 긴 문자열, 넓은 표, 색상 변경을 확인한다.
- 게시글과 iframe의 가로 넘침, 폭 축소, 잘림, 중복 스크롤, 버튼 줄바꿈을 확인한다.
- 발견한 문제는 수정 후 같은 화면 크기와 상태에서 다시 검증한다.
- 공개 모바일 홈 최종 검수에서 검색 폼이 Blogger의 `content-box`를 상속해 14px 넘치는 현상을 발견했으며, 검색 폼에 `border-box`를 명시해 가로 스크롤을 제거한다.

## 실행 및 배포 명령

1. `node scripts/build-tool-catalog.mjs`: 각 도구의 최신 버전으로 전체 카탈로그를 다시 만든다.
2. `node scripts/validate-tool-releases.mjs`: JSON 구문과 최신 HTML·패치 노트 경로를 검증한다.
3. `node scripts/blogger-update-tool-posts.mjs json-tools base64-tools qr-barcode-generator`: 공개 게시글 제목과 본문을 최신 버전으로 갱신한다.
4. Git `main` 브랜치에 관련 변경을 한 커밋으로 저장하고 푸시한 뒤 GitHub Pages 배포 성공을 확인한다.
5. 공개 홈과 세 게시글을 데스크톱·모바일에서 다시 열어 최종 디자인 검수를 기록한다.
