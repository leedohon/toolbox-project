# 텍스트 변환 확장과 게시글 유입 개선

## 요청

- 텍스트 인코더·디코더에 ASCII와 Unicode를 포함한 실용적인 변환 형식을 추가한다.
- 모든 도구 게시글에 검색·탐색용 해시태그를 일관되게 제공한다.
- 게시글 진입 시 크게 표시되는 `오픈월드 툴박스` 사이트 헤더를 정리한다.

## 구현 범위

- 텍스트 변환 형식: Base64, URL, ASCII 10진 코드, Unicode 코드 포인트, UTF-8 16진수, UTF-8 2진수
- 형식이 5개 이상이므로 공통 UX 원칙에 따라 형식 선택은 셀렉트로 제공한다.
- ASCII는 0~127 범위만 허용하고 한글 등 범위 밖 문자는 Unicode 사용을 안내한다.
- Unicode는 `U+XXXX` 형식의 코드 포인트를 사용하고 보조 평면 문자도 올바르게 처리한다.
- 16진수와 2진수는 UTF-8 바이트를 기준으로 변환한다.
- 공통 해시태그와 도구별 해시태그는 `toolbox/post-tags.json` 한 곳에서 관리한다.
- 생성 스크립트가 해시태그 링크를 모든 게시글 본문에 삽입하고 Blogger API의 라벨에도 같은 값을 반영한다.
- Blogger 게시 제목의 `[초간단 툴박스] 도구명` 브랜드 형식은 유지한다.
- Blogger 기본 사이트 헤더 대신 메인 화면의 Openworld Blog 소개문과 도구 검색 UI를 게시글 상단에도 공통으로 표시한다.
- 검색어가 없을 때는 도구 링크를 숨기고, 입력 뒤에만 JSON 카탈로그의 해시태그·20~30개 키워드·유사어를 기준으로 일치하는 링크를 표시한다.

## 검증

- 각 텍스트 형식의 왕복 변환과 잘못된 입력 오류를 확인한다.
- KOR / ENG에서 새 라벨·상태·오류가 모두 번역되는지 확인한다.
- 모바일과 PC에서 셀렉트, 입력·결과, 해시태그, 컴팩트 사이트 헤더, 가로 넘침을 확인한다.
- 기존 공개 게시글 전체에 해시태그, 새 제목 형식, Blogger 라벨이 반영되는지 확인한다.
