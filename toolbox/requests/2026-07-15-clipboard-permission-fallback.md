# 클립보드 권한 차단 대응

## 요청

- 편가르기 결과 복사에서 iframe 권한 정책으로 Clipboard API가 차단되어도 오류가 노출되지 않게 한다.
- 가능한 경우 호환 복사 방식을 사용하고, 자동 복사가 모두 차단되면 사용자가 직접 복사할 결과를 화면에 제공한다.
- 같은 공통 복사 함수를 사용하는 숫자뽑기에도 동일하게 적용한다.
- 불가능한 기능은 제거할 수 있으나, 이번 문제는 안전한 대체 동작이 가능하므로 결과 복사 기능을 유지한다.

## 구현 결정

- 최신 Clipboard API를 먼저 시도한다.
- 권한 거부 시 브라우저 호환 복사 방식을 시도한다.
- 두 방식이 모두 실패하면 Promise 오류를 발생시키지 않고 `false`를 반환한다.
- 호출 화면은 직접 선택할 수 있는 읽기 전용 결과 상자를 표시하고 쉬운 안내 문구를 제공한다.
- 편가르기와 숫자뽑기는 `minimum` 수정으로 각각 `0.3.1v`를 릴리스한다.

## 검증

- Clipboard API 권한 거부를 강제로 재현해 콘솔의 처리되지 않은 Promise 오류가 없는지 확인한다.
- 호환 복사 성공과 전체 자동 복사 실패 시 직접 복사 상자가 표시되는지 확인한다.
- KOR / ENG 문구와 모바일·데스크톱 레이아웃을 확인한다.
