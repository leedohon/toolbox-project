# 기존 도구 단순 처리 흐름 개선

## 사용자 요청

- 기존 기능에서 복잡한 확장보다 반복 조작을 줄이는 단순 개선점을 선정해 구현한다.
- 에이전트가 개선 대상을 선정하고 구현, 게시글 수정, 검증, 커밋, 푸시와 공개 배포까지 완료한다.
- 완료 후 각 게시글과 iframe 도구의 데스크톱·모바일 디자인을 직접 확인한다.

## 선정 기준

- 기존 핵심 기능을 바꾸지 않고 클릭이나 입력 단계를 줄인다.
- 한 화면의 목적을 흐리는 설정이나 가입 요구를 추가하지 않는다.
- 브라우저 안에서 즉시 처리할 수 있고 안전성과 정확성이 명확한 기능만 선택한다.
- KOR / ENG 공통 언어 모듈과 기존 버튼 체계를 재사용한다.

## 확정 개선

1. QR · 바코드 생성기
   - 입력, 유형, QR 스타일, 크기, 색상, 바코드 텍스트 표시를 기본값으로 한 번에 되돌리는 `기본값으로 초기화` 버튼을 추가한다.
   - 다음 버전: `0.5.0v` (`minor`)
2. JSON 도구
   - Excel 표 변환까지 바로 시험할 수 있는 객체 배열 예제를 입력하는 `예제 JSON 불러오기` 버튼을 추가한다.
   - 다음 버전: `0.6.0v` (`minor`)
3. Base64 도구
   - 현재 결과를 다음 입력으로 옮기고 인코딩/디코딩 반대 모드로 전환하는 `입력·결과 바꾸기` 버튼을 추가한다.
   - 결과가 없으면 실행하지 않고 가까운 상태 영역에 이유를 안내한다.
   - 다음 버전: `0.6.0v` (`minor`)
4. 숫자뽑기
   - `로또 6/45`와 `주사위 1~6` 빠른 설정 버튼을 추가해 범위·개수·중복·정렬 값을 한 번에 적용하고 즉시 추첨한다.
   - 다음 버전: `0.2.0v` (`minor`)

## 제외 범위

- 나머지 도구는 현재 핵심 흐름이 이미 짧고, 이번에 임의 기능을 더하면 화면 목적이 흐려질 수 있어 변경하지 않는다.
- 로그인, 저장 계정, 서버 처리, 기록 동기화는 추가하지 않는다.

## 검증 및 배포 명령

```text
node scripts/build-tool-catalog.mjs
node scripts/validate-tool-releases.mjs
node scripts/blogger-update-tool-posts.mjs qr-barcode-generator json-tools base64-tools number-picker
git status --short
git add <이번 작업 파일>
git commit -m "feat(toolbox): streamline common tool actions"
git push origin main
```

## 완료 기준

- 각 보조 동작이 KOR/ENG에서 같은 기능을 제공한다.
- 빈 결과, 모드 전환, 설정 초기화와 빠른 설정이 기존 결과를 깨뜨리지 않는다.
- 375px 모바일과 1280px 데스크톱에서 버튼 잘림, 가로 넘침, 비정상적인 간격이 없다.
- 실제 브라우저 점검 결과를 각 새 버전의 `patch-notes.json`에 기록한다.
