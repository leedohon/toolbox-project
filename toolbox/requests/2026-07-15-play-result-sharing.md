# 놀이 도구 결과 코드 공유와 PNG 저장

## 사용자 요청

- 놀이 도구 결과를 다른 사람에게 쉽게 공유할 수 있게 한다.
- 결과 코드를 복사하고, 받은 코드를 입력하면 같은 결과 또는 같은 시드를 볼 수 있게 한다.
- IT 지식이 없는 사용자도 설명 없이 사용할 수 있는 흐름으로 만든다.
- 좋은 공유 방법을 우선 적용하고, 불편하거나 불가능할 때 PNG 저장을 마지막 수단으로 제공한다.

## 선택한 방식

- 시드만 전달하는 방식보다 **입력과 확정 결과를 함께 담은 결과 코드**를 사용한다.
- 이유: 구현 변경이나 난수 알고리즘 차이와 관계없이 받은 사람이 정확히 같은 결과를 볼 수 있고, 사용자가 시드 개념을 알 필요가 없다.
- 코드는 `TB1-`로 시작하는 URL-safe 문자열이며 버전, 도구 식별자와 필요한 결과 데이터만 포함한다.
- 이름과 후보는 코드 안에 들어가므로 공유 전에 사용자가 내용을 확인하도록 짧은 개인정보 안내를 제공한다. 코드는 서버로 전송하거나 저장하지 않는다.
- 다른 도구의 코드, 손상된 코드, 지나치게 긴 코드는 실행하지 않고 입력 영역 가까이에서 쉬운 문장으로 안내한다.

## 공통 UX

1. 결과가 만들어진 뒤 `결과 코드 복사`를 누른다.
2. 받은 사람은 같은 도구의 `받은 결과 코드` 입력칸에 붙여 넣고 `같은 결과 열기`를 누른다.
3. 코드가 정상이라면 입력값과 결과 화면을 함께 복원한다.
4. 별도 공유나 보관이 필요하면 `PNG 저장`으로 제목·설정·결과가 들어간 결과 카드를 저장한다.
5. 기술 용어 대신 `결과 코드`, `같은 결과`, `받은 코드`라는 표현만 사용한다.

## 도구별 저장 데이터와 버전

- 사다리게임 `0.3.0v`: 참가자, 결과, 사다리 가로선과 최종 연결 결과
- 돌림판 `0.3.0v`: 후보 목록과 당첨 항목
- 숫자뽑기 `0.3.0v`: 범위, 개수, 중복·정렬 설정과 뽑힌 숫자
- 다중팀 편가르기 `0.3.0v`: 팀 이름과 확정된 팀별 참가자
- 변경 등급은 네 도구 모두 `minor`다.

## PNG 출력 원칙

- 외부 캡처 서버나 계정을 사용하지 않고 현재 브라우저에서 Canvas로 생성한다.
- 화이트·스카이블루 공통 카드, 도구명, 설정 요약, 결과 목록과 생성 시각을 포함한다.
- 긴 이름은 자동 줄바꿈하고 결과 전체가 잘리지 않도록 카드 높이를 계산한다.

## 안전 및 제한

- 코드 최대 길이는 20,000자로 제한한다.
- 복원 데이터는 도구별 인원·항목·숫자 범위 제한을 다시 검증한다.
- HTML이나 명령을 실행하지 않고 순수 JSON 데이터로만 해석한다.
- 클립보드 권한이 없으면 코드 입력칸을 선택해 사용자가 직접 복사할 수 있게 한다.


## 완료 기준

- 네 도구에서 결과 코드 복사와 같은 결과 복원이 정확히 동작한다.
- 다른 도구 코드와 손상된 코드가 안전하게 거부된다.
- PNG 파일이 결과 전체를 포함해 생성된다.
- KOR/ENG, 375px 모바일과 1280px 데스크톱에서 입력·버튼·상태 문구가 잘리지 않는다.
- 실제 디자인 검수 결과를 각 새 버전의 `patch-notes.json`에 기록한다.
