# 전역 게시글 누적 패치노트

## 요청

- 모든 공개 도구 게시글의 자주 묻는 질문 영역에 `패치노트` 항목을 추가한다.
- 패치노트 안에는 도구의 버전과 해당 버전 변경 목록을 표시한다.
- 최신 버전을 위에 두고 기존 버전도 삭제하지 않은 채 계속 누적한다.
- 앞으로 새 버전을 만들 때 게시글의 누적 패치노트도 함께 갱신한다.
- 각 글 하단에 도구의 용도와 처리 방식을 설명하는 상세 설명을 추가한다.
- FAQ는 도구별로 최소 4개를 제공하고 실제 사용자 질문만 다룬다.
- Base64 실행 화면을 다른 도구와 같은 스카이블루 공통 베이스 스타일로 통일한다.
- 도구 검색 복귀 링크는 게시글 좌측 하단 플로팅 버튼으로 제공한다.
- 게시글 초안 작성과 릴리스 절차를 재사용 가능한 스킬과 스크립트로 자동화한다.
- 모바일 터치 환경에서는 입력칸에 자동 포커스를 주지 않아 키패드가 임의로 열리지 않게 한다.

## 구현 결정

- 프로젝트의 기존 버전 형식인 `[major].[minor].[minimum]v`를 유지한다. 화면에도 `0.1.0v`, `0.1.1v`처럼 표시한다.
- 패치노트는 FAQ 안의 접이식 항목으로 제공해 글의 정보량은 늘리되 핵심 도구 사용을 방해하지 않게 한다.
- 표시 내용은 각 버전의 `patch-notes.json`을 단일 원본으로 사용한다.
- `scripts/build-post-patch-notes.mjs`가 모든 버전을 최신순 정적 HTML로 생성해 Blogger 본문과 최신 릴리스 HTML에 반영한다.
- `toolbox/post-content.json`을 상세 설명과 사용자 FAQ의 단일 원본으로 사용하고 `scripts/build-tool-posts.mjs`가 게시글 전체 공통 영역을 생성한다.
- FAQ에는 사용 방법, 입력과 결과, 개인정보 처리, 오류 해결처럼 사용자가 실제로 궁금해할 내용만 둔다. 게시글 갱신 방식 같은 운영자용 문구는 제거한다.
- 새 `toolbox-post-draft` 스킬은 긴 규칙을 복제하지 않고 저장소의 MD, 중앙 콘텐츠와 자동화 명령만 연결한다.
- 모든 embed는 `assets/toolbox-ux.js`의 안전 포커스 헬퍼를 사용한다. 767px 이하 또는 coarse pointer에서는 자동 포커스와 자동 선택을 생략한다.
- 최신 공개 게시글에 누적 패치노트가 없거나 버전이 빠지면 릴리스 검증을 실패시킨다.

## 이번 릴리스

- 모든 공개 도구에 누적 패치노트 항목을 처음 적용한다.
- 실행 기능은 바뀌지 않으므로 각 도구의 `minimum` 버전만 한 단계 올린다.
- Blogger의 기존 공개 글을 중복 생성하지 않고 최신 본문으로 갱신한다.

## 검증

- 모든 버전과 변경 목록이 최신순으로 출력되는지 확인한다.
- 패치노트를 펼쳤을 때 모바일과 PC에서 가로 넘침, 비정상적인 폭 축소, 긴 목록 잘림이 없는지 확인한다.
- FAQ의 기존 항목과 iframe 높이 조절 기능이 그대로 동작하는지 확인한다.
