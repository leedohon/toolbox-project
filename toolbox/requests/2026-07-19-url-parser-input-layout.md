# URL 분석·쿼리 편집기 입력 UI 정리

## 사용자 요청

- 공개 URL 분석·쿼리 편집기의 입력 상자가 엉성하게 보이는 문제를 개선한다.

## 확인한 원인

- 정적 구성요소 입력과 동적 쿼리 입력에 `type="text"` 속성이 없어 공통 입력 CSS의 속성 선택자와 일치하지 않았다.
- 모바일 쿼리 행에서 삭제 버튼이 입력칸과 같은 전체 폭을 차지해 행의 위계가 흐트러졌다.

## 확정 구현

- 모든 한 줄 URL 편집 입력에 명시적인 `type="text"`를 적용한다.
- 읽기 전용 구성요소는 편집 가능한 경로·해시와 시각적으로 구분한다.
- 모바일 쿼리 행은 번호, 이름, 삭제, 값 순서가 분명한 2단 배치로 정돈한다.
- 375px와 1280px에서 입력 높이·폭·가로 넘침·키보드 흐름을 검증한다.

## 변경 등급

- `minimum`: 기능 결과는 유지하고 입력 UI와 반응형 배치만 수정한다.
