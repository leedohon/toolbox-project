# Skill·Markdown 점진 최적화

## 요청

- 모든 자동 워크플로에서 Skill과 Markdown의 읽기·판단·반복 수정 비용을 계속 줄인다.
- 한 번에 전체 문서를 다시 쓰지 않고 작업 중 발견한 개선점을 공용으로 보관한 뒤 관련 파일을 수정할 때 함께 적용한다.
- 1회 최적화의 감소 목표는 50% 이상으로 두되 과도한 토큰을 쓰지 않는다.
- 정책 버전과 일괄 적용 비용을 줄이는 공통 장치를 둔다.

## 적용

- `toolbox/automation/knowledge-optimization.json`을 단일 원본으로 사용한다.
- 정책은 `major.minor.patch`로 버전 관리하고 대기열 항목 추가만으로는 버전을 올리지 않는다.
- 워크플로당 관련 있는 최적화는 최대 1건, 전체 실행 예산의 5% 이내에서 처리한다.
- 관련 없는 개선은 구현하지 않고 중복을 제거해 대기열에 기록한다.
- 같은 측정 기준의 전후 증거가 있을 때만 실제 감소율을 기록하며, 증거가 없으면 `unmeasured`로 남긴다.
- 기존 전체 문서와 스킬을 소급 일괄 수정하지 않는다.
