# 2026-07-19 데일리 라이트 워크플로

## 확정 요청

- 작업 종류: 데일리 라이트 자동 작업 처리
- 버전 단계: 영향도에 따라 기존 도구는 minimum, 신규 도구는 0.0.1v
- Blogger 게시: 완료된 변경만 공개 반영
- Git·배포: `main` 커밋·푸시와 GitHub Pages 공개 확인
- 종료 예약: 사용하지 않음

## 공개 도구 개선 3건

1. `password-generator`: 문자 그룹을 모두 해제했을 때 이전 비밀번호가 복사될 수 있는 상태를 제거하고, 생성 결과가 없으면 복사 버튼을 비활성화한다. 강도 설명은 보조기술에서도 바로 읽을 수 있게 유지한다.
2. `image-resizer-compressor`: 드롭 영역 안에 키보드로 실행 가능한 파일 선택 버튼을 제공하고, 변환 파일명은 원본 이름을 보존해 저장한다.
3. `color-converter-contrast`: 3자리 HEX 입력을 지원하고, 입력 중간 상태에서는 성급한 오류를 표시하지 않는다. UI 구성요소 3:1 판정과 복사 대체 안내를 추가한다.

## 신규 페이지 1건

- `percentage-calculator`: 비율값, 전체의 몇 퍼센트, 증감률을 한 페이지의 명확한 모드 선택으로 계산한다.
- 입력: 모드별 두 수치
- 결과: 계산식, 결과값, 복사 가능한 요약
- 차이점: 기존 날짜·화면 비율·CSS 계산기와 달리 일반적인 백분율 관계 계산에 집중한다.
- 구현 판단: 서버나 외부 라이브러리 없이 브라우저에서 독립 구현 가능하며 예상 복잡도는 낮음이다.

## 추가 아이디어 3건

1. 이미지 메타데이터 확인기
   - 문제: 이미지의 픽셀 크기와 파일 형식을 열기 전에는 확인하기 번거롭다.
   - 입력·결과: 이미지 파일 → 크기, 형식, 용량, 가능한 메타데이터 요약.
   - 차이: 이미지 변환 없이 읽기 전용 검사에 집중한다.
   - 구현 가능성·복잡도: 브라우저 독립 구현 가능, 중간. EXIF 범위 결정이 필요해 이번에는 제안으로 유지한다.
2. 시간대 변환기
   - 문제: 여러 지역의 회의 시간을 동시에 맞추기 어렵다.
   - 입력·결과: 날짜·시각·기준 시간대 → 선택 지역별 현지 시각.
   - 차이: Unix 시간 변환보다 사람이 사용하는 지역 시간 비교에 집중한다.
   - 구현 가능성·복잡도: `Intl` 기반 독립 구현 가능, 중간. 시간대 목록 UX 검토가 필요하다.
3. JSON 경로 조회기
   - 문제: 큰 JSON에서 중첩 값을 빠르게 찾기 어렵다.
   - 입력·결과: JSON과 경로 → 해당 값과 오류 위치.
   - 차이: 기존 JSON 정리·표 변환이 아니라 특정 경로 조회에 집중한다.
   - 구현 가능성·복잡도: 브라우저 구현 가능, 중간. 지원할 경로 문법을 먼저 확정해야 한다.

## 공통 개선 제안 3건

1. 파일 드롭 영역의 Enter·Space·드래그 상태를 한 헬퍼로 통합한다. 이번에는 단일 대상에서 먼저 검증하고 반복 적용 근거가 생기면 공통화한다.
2. 숫자 두 개를 받는 계산기의 유효성 검사와 KOR / ENG 오류 문구를 공통 헬퍼로 분리한다. 계산식별 범위 차이를 설계한 뒤 적용한다.
3. 클립보드 차단 시 대체 영역 표시와 상태 안내를 공통 함수로 통합한다. 기존 도구별 결과 구조 차이를 점검한 뒤 적용한다.

## 검증 기준

- 375×812 모바일과 1280×900 데스크톱에서 가로 넘침, 잘림, 중복 스크롤이 없어야 한다.
- 초기·정상·오류·복사 대체 상태와 KOR / ENG 문구를 확인한다.
- `IFRAME_HEIGHT_STALE`, `HIDDEN_GRID_OVERRIDE`, `SHARED_ASSET_CACHE_STALE` 회귀 검사를 적용한다.
- 파일 선택 버튼은 키보드로 실행할 수 있고 드래그 앤 드롭과 같은 검증 흐름에 도달해야 한다.
- 신규 도구는 정의, embed, 게시글, 버전, 패치노트, 카탈로그, 번역, AI 인덱스를 모두 갖춘다.
