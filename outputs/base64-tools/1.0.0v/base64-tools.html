  1. 텍스트 인코딩, 진법, 유니코드 분석, HTML 엔티티, 로마 숫자 중 필요한 기능을 선택합니다.
  2. 선택한 모드에 맞는 원문·코드·숫자를 입력합니다.
  3. 변환 또는 분석 결과를 확인하고 필요한 값을 복사합니다.

인코딩·문자 변환 도구 상세 설명

인코딩·문자 변환 도구는 Base64·URL·ASCII·UTF-8 변환, 정수 진법 변환, 유니코드 문자 분석, HTML 엔티티 변환과 로마 숫자 변환을 한 화면에서 선택해 사용하도록 통합했습니다. 텍스트 표현 방식이나 숫자 표기를 확인할 때 여러 게시글을 오가지 않고 필요한 모드만 바꿔 작업할 수 있습니다.

텍스트 인코딩 모드는 일반·URL-safe Base64와 파일 data URL을 지원하고, 진법 모드는 2진수·8진수·10진수·16진수 정수를 정밀도 손실 없이 변환합니다. 유니코드 분석은 문자별 코드 포인트와 UTF-16·HTML 표기를 보여 주며, HTML 엔티티와 로마 숫자 모드는 각각 특수문자와 1~3999 범위의 표준 숫자 표기를 양방향으로 바꿉니다.

모든 입력과 파일은 현재 브라우저 안에서만 처리되어 별도 서버로 전송되지 않습니다. 형식에 맞지 않는 값이나 지원 범위를 벗어난 숫자는 가까운 위치에서 안내하며, 원문을 수정해 다시 실행하거나 초기화한 뒤 다른 모드를 선택할 수 있습니다.

자주 묻는 질문

어떤 기능을 한곳에서 사용할 수 있나요?

Base64·URL·ASCII·UTF-8 인코딩, 2·8·10·16진수 변환, 유니코드 문자 분석, HTML 엔티티 변환과 로마 숫자 변환을 지원합니다.

문자 코드와 진법 변환은 어떻게 다른가요?

문자 코드는 텍스트를 바이트나 코드 포인트로 표현하고, 진법 변환은 같은 정수 값을 2진수·8진수·10진수·16진수 표기로 바꿉니다.

HTML 특수문자와 로마 숫자도 되돌릴 수 있나요?

네. HTML 문자를 엔티티로 바꾸거나 원문으로 되돌릴 수 있고, 1~3999의 아라비아 숫자와 표준 로마 숫자도 서로 변환할 수 있습니다.

한글이나 이모지 정보를 자세히 보려면 어떤 모드를 쓰나요?

유니코드 문자 분석을 선택하면 문자별 코드 포인트, UTF-16 단위, UTF-8 바이트와 HTML 표기를 확인할 수 있습니다.

입력 형식 오류가 나오면 어떻게 하나요?

선택한 모드의 예시와 지원 범위를 확인해 공백, 접두사, 잘못된 문자나 숫자 범위를 고친 뒤 다시 실행하세요. 초기화하면 기본 예시로 돌아갑니다.

입력한 텍스트나 파일이 서버로 전송되나요?

아니요. 모든 변환과 분석은 현재 브라우저 안에서만 진행됩니다. Base64는 암호화가 아니므로 민감한 정보를 보호하는 용도로 사용하지 마세요.
